# 图书馆记忆报告 H5 设计记录

## 参考的开源项目模式

- Swiper: 参考其移动端纵向分屏、触摸/滚轮/键盘导航、Autoplay 与硬件加速转场思路。本项目继续保持单文件原生实现，使用 `translate3d` 与逐页 `is-active` 状态，避免新增构建依赖。
- CountUp.js: 参考其“元素进入当前页后再启动数字缓动”的体验。本项目保留 `data-count` 的可见页触发计数，避免所有数字在加载时一次性跳完。
- howler.js: 参考其 HTML5 Audio/Web Audio 兼容思路和“移动端必须用户交互后播放”的限制。本项目用原生 `<audio>` 播放 FLAC 背景音乐，用 Web Audio 生成轻量 UI 音效。

## 本轮实现原则

- 每页只突出一个主结论：年度峰值、最红月份、最重学期、关键岔路分别拆成重点页。
- 保留密集数据页：年度总表、月份热力墙、学期编年史、关键阶段信号表作为总结页。
- 控件不遮挡正文：顶部工具栏固定，正文统一增加顶部安全留白。
- 叙事动效必须是逐条揭晓：kicker、标题、说明、关键数字、卡片和数据行按顺序弹出，节奏要明显偏慢，不能只做整页淡入。
- 自动翻页默认开启，约 20 秒一页；当前页内容弹出完成后再开始计时，手动翻页会重置计时。
- 音乐默认开启并尝试播放；如果浏览器拦截自动播放，则在用户第一次点击、滑动或按键后续播。左上角曲名用于切换下一首，右上角唱片用于开关静音。
- “生成完整长图”必须导出所有可见分镜页，不只导出最后诊断小票；导出时逐页渲染后再拼接，避免移动端一次渲染超高 DOM 变成空白图。
- 分享按钮保存公开报告数据，生成 `https://your-domain.example/{学号}` 链接；公开报告不包含保存口令。
- 总时长排行榜只统计用户主动保存或分享后的报告，按 `global_stats.total_hours` 排序。

## 页面拆分规则

- 座位领地拆成两页：第一页只讲行动半径、主基地和锐评；第二页展示所有去过楼层/空间的占领清单。
- 期末周心电图拆成两页：第一页展示预警、期末总预约和 24 小时心电图；第二页展示压力月份脉冲，避免图表和列表挤在同一屏。
- 期末周第一页如果文案讨论闭馆线，图表必须使用签退/离馆时间分布；期末月份页只讨论补天月份和压力峰值，不混用闭馆线吐槽。
- 结尾包括诊断报告页和毕业快乐页；毕业快乐页使用同源静态图片 `docs/assets/graduation-happy.png`。
